🐛 Phần 2: Code Review & Phân Tích Lỗ Hổng

2.1 Bugs & Code Smells

🔴 BUG-CRITICAL: Logic trùng lặp Controller ↔ Service
OrdersController.CreateOrder() chứa 170 dòng business logic inline mà KHÔNG gọi IOrderService. Cùng logic tồn tại song song → sửa 1 chỗ gây inconsistency.
Tương tự: KitchenController và PaymentsController đều bypass Service layer.
🔴 BUG: Hardcoded status strings trong Controller
OrdersController dùng "paid", "cancelled" (hardcode), trong khi OrderService dùng OrderStatus.Paid constants → Dễ typo, khó maintain.
🟡 BUG: CancelOrderAsync so sánh case-sensitive
cancellableStatuses.Contains(order.Status?.ToLower()) — nên dùng StringComparer.OrdinalIgnoreCase cho an toàn.
🟡 Code Smell: Entity trả thẳng ra API (risk circular ref + data leak) • LoginDto/UpdateUserDto inline trong Controller • Không có Global Exception Handler • GetDashboardStats trả object thay vì typed DTO
