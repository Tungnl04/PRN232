🐛 Phần 2: Những Lỗi Cần Sửa

2.1 Lỗi Code

🔴 Lỗi 1: Code bị trùng lặp (quan trọng nhất!)
OrdersController.CreateOrder() viết lại toàn bộ logic 170 dòng thay vì gọi OrderService.
→ Cùng 1 logic tồn tại ở 2 chỗ. Sửa 1 chỗ quên chỗ kia = bug.
Cách sửa: Controller chỉ gọi _orderService.CreateOrAppendOrderAsync(), xóa logic inline.
🔴 Lỗi 2: Dùng chuỗi "cứng" thay vì hằng số
Controller viết "paid", "cancelled" trực tiếp → dễ gõ sai.
Service đã dùng OrderStatus.Paid đúng cách.
Cách sửa: Thay tất cả string cứng bằng StatusConstants.
🟡 Lỗi 3: Trả Entity gốc ra API
Một số endpoint trả thẳng Entity (Product, Order) → có thể lộ dữ liệu không mong muốn.
Cách sửa: Tạo DTO riêng cho response.
🟡 Lỗi 4: Không có xử lý lỗi chung
Nếu database lỗi → stack trace hiện ra cho người dùng.
Cách sửa: Thêm Exception Handling Middleware.
🟡 Lỗi 5: Đặt Combo không trừ kho món con
Khi khách đặt Combo, chỉ check combo.Available nhưng không trừ inventory các product bên trong.
Cách sửa: Duyệt qua ComboItems → trừ kho từng product.
🟢 Lỗi nhỏ: LoginDto nằm trong AuthController file (nên chuyển vào folder DTOs) • Bàn chuyển "available" ngay sau thanh toán (nên có trạng thái "cleaning")

3.2 SignalR Hoạt Động Như Thế Nào?

SignalR = công nghệ realtime của .NET. Thay vì client phải gọi API liên tục (polling), server sẽ tự đẩy thông báo đến client khi có thay đổi.
sequenceDiagram participant Guest as 📱 Khách participant API as 🖥️ Server participant Hub as 📡 SignalR Hub participant Kitchen as 👨‍🍳 Bếp participant Staff as 💁 Staff Guest->>API: Đặt món API->>Hub: Gửi "Có đơn mới!" Hub-->>Kitchen: 🔔 Hiện đơn trên màn hình bếp Hub-->>Staff: 🔔 Hiện thông báo Kitchen->>API: Đánh dấu "Nấu xong" API->>Hub: Gửi "Món đã xong!" Hub-->>Guest: 🎉 Món sắp được bưng ra! Hub-->>Staff: 📢 Cần bưng món bàn X

3.3 Tạo QR Cho Từng Bàn — Cách Hoạt Động

Mỗi bàn có 1 mã QR riêng. Khách quét QR → mở trang menu của đúng bàn đó → đặt món → server biết đơn thuộc bàn nào.
sequenceDiagram participant Admin as 🔑 Admin participant API as 🖥️ Server participant QR as 🔲 Mã QR participant Guest as 📱 Khách Admin->>API: Tạo/Refresh QR cho Bàn 5 API->>API: Sinh token ngẫu nhiên, lưu vào order_table.qr_code_token API-->>Admin: URL = /menu?table=5&token=abc123 Admin->>QR: In mã QR chứa URL này dán lên bàn Guest->>QR: Quét mã QR bằng điện thoại QR-->>Guest: Mở trang menu bàn 5 Guest->>API: Đặt món kèm tableId=5 + token=abc123 API->>API: Kiểm tra token khớp → OK → tạo đơn

3.4 Thanh Toán VNPay (Sandbox)

sequenceDiagram participant Staff as 💁 Staff participant API as 🖥️ Server participant VNPay as 💳 VNPay Sandbox Staff->>API: Yêu cầu tạo link thanh toán API->>VNPay: Gửi thông tin đơn hàng VNPay-->>API: Trả về URL thanh toán API-->>Staff: Hiện mã QR thanh toán Note right of VNPay: Khách quét QR → thanh toán VNPay->>API: Gọi callback "đã thanh toán OK" API->>API: Cập nhật đơn = paid, mở bàn
